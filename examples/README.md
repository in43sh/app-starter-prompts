# Examples

These examples show the kind of output each prompt should produce when the interview goes well.

Keep these examples structurally aligned with the current prompt docs. If a prompt's required output sections change, update the corresponding example in the same PR.

- `00-kickoff-web-app/` contains a fictional project kickoff for a neighborhood marketplace web app.
- `00-kickoff-api/` contains a fictional backend kickoff for a package-tracking notification API.
- `01-feature-spec/` contains a fictional feature spec for saved search alerts in an existing app.
- `02-claude-md/` contains a fictional `CLAUDE.md` for a SaaS monorepo.

The examples are intentionally concise, but they should still feel implementation-ready rather than aspirational.
